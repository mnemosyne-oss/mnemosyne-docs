# Multi-Agent Memory

Use separate Mnemosyne instances to coordinate memory across a team of agents.

<Callout type="warning" title="Same-Machine Only">
Mnemosyne's multi-agent sharing works via the same SQLite database file on the same filesystem. Cross-machine sharing requires manual export/import or a future MCP server. If you need HTTP-based memory sharing across machines, consider Hindsight self-hosted instead.
</Callout>

## Separate Database Instances

Each agent gets its own `Mnemosyne` instance. Use separate `db_path` values for isolation, or share a path for coordination:

<CodeBlock code={`from mnemosyne import Mnemosyne

# Each agent has its own isolated database
agent_a = Mnemosyne(
    session_id="agent-a",
    db_path="/data/agent-a/memories.db",
)

agent_b = Mnemosyne(
    session_id="agent-b",
    db_path="/data/agent-b/memories.db",
)

# Agent A stores an observation
agent_a.remember(
    content="The API rate limit is 1000 req/min.",
    source="observation",
    importance=0.9,
)

# Agent B cannot see Agent A's memories
results = agent_b.recall("API rate limit")  # Returns nothing from agent-a's DB
`} language="python" />

## Shared Memory Pattern

To share knowledge between agents on the same machine, point them at the same database file with different `session_id` values:

<CodeBlock code={`from mnemosyne import Mnemosyne

# Both agents share the same database file
agent_a = Mnemosyne(
    session_id="agent-a",
    db_path="/shared/team-memory.db",
)

agent_b = Mnemosyne(
    session_id="agent-b",
    db_path="/shared/team-memory.db",
)

# Agent A stores team knowledge
agent_a.remember(
    content="The API rate limit is 1000 req/min.",
    source="infrastructure-check",
    importance=0.9,
    scope="global",  # Visible to all sessions
)

# Agent B can recall shared knowledge
results = agent_b.recall("API rate limit")
`} language="python" />

## Private vs Shared Context

Use `scope` to control what is private vs shared:

<CodeBlock code={`from mnemosyne import Mnemosyne

agent_a = Mnemosyne(session_id="agent-a", db_path="/shared/team-memory.db")

# Private to agent-a's session (default)
agent_a.remember(
    content="My current subtask: validate user inputs.",
    scope="session",
    importance=0.5,
)

# Shared with all sessions in this database
agent_a.remember(
    content="Deployment target is us-east-1 region.",
    scope="global",
    importance=0.8,
)
`} language="python" />

## Cross-Machine Sharing

For sharing memory across different machines, use export/import:

<CodeBlock code={`# On machine A: export shared memories
agent_a.export_to_file("/shared/team-memory-export.json")

# On machine B: import
agent_b.import_from_file("/shared/team-memory-export.json")
`} language="python" />

<Callout type="info" title="Future: MCP Server">
A lightweight MCP (Model Context Protocol) server for Mnemosyne BEAM is planned. This will enable cross-machine memory sharing without manual export/import.
</Callout>

## Deployment

Use a shared volume or network filesystem:

- **Docker**: Named volume mounted to all agents
- **Kubernetes**: Shared PVC
- **Fly.io**: Same volume mounted to multiple machines

<Callout type="warning" title="SQLite Concurrency">
SQLite handles multiple readers well but prefers single-writer. For high-write multi-agent scenarios, consider WAL mode or connection pooling.
</Callout>
