# Monitoring

Keep your Mnemosyne deployment healthy with built-in metrics and health checks.

## Health Check

Mnemosyne does not expose an HTTP server. Use CLI commands to verify health:

<CodeBlock code={`# Check memory stats
hermes mnemosyne stats

# Verify database integrity
sqlite3 ~/.hermes/mnemosyne/data/mnemosyne.db "PRAGMA integrity_check"
`} language="bash" />

For programmatic health checks, use the `mnemosyne_diagnose` MCP tool, which returns database stats, table counts, and integrity status.

## Metrics

Retrieve memory metrics via the CLI or MCP tool:

<CodeBlock code={`# Global stats across all Hermes profiles
hermes mnemosyne stats --global

# Per-profile stats (run from a project directory or specify --profile)
hermes mnemosyne stats
`} language="bash" />

The `mnemosyne_diagnose` MCP tool returns structured metrics including:
- `working_memory`, `episodic_memory`, and `scratchpad` table counts
- Database size and integrity status
- Embedding model info
- Version info (3.4.0)

## Key Metrics

Retrieve these with `hermes mnemosyne stats` or `mnemosyne_diagnose`:

- **working_memory**: Items currently in active working memory (max controlled by `MNEMOSYNE_WM_MAX_ITEMS`, default 10000)
- **episodic_memory**: Total consolidated episodes across all tiers
- **scratchpad**: Temporary in-flight entries during sleep/consolidation
- **Database size**: `du -sh ~/.hermes/mnemosyne/data/mnemosyne.db`
- **Last sleep / consolidation time**: Reported in `hermes mnemosyne stats` output
- **Vector index**: sqlite-vec virtual table (`vec_episodes`) for embedding search

## Logging

Mnemosyne uses Python's standard `logging` module. To enable debug logging:

<CodeBlock code={`import logging
logging.getLogger("mnemosyne").setLevel(logging.DEBUG)
`} language="python" />

Log output includes component-level detail without requiring special configuration. Key loggers:
- `mnemosyne.beam` — memory tiering, sleep/consolidation
- `mnemosyne.embeddings` — model loading and generation
- `mnemosyne.dr.recovery` — backup and integrity checks

## Health Check Scripts

For automated monitoring, use cron or a scheduled job that runs health checks:

<CodeBlock code={`#!/bin/bash
# mnemosyne-health-check.sh
set -e

# Integrity check
sqlite3 ~/.hermes/mnemosyne/data/mnemosyne.db \\
  "PRAGMA integrity_check" | grep -q "ok" || exit 1

# Check working memory hasn't overflowed
hermes mnemosyne stats --json | python3 -c "
import json, sys
d = json.load(sys.stdin)
if d['working_total'] > 9500:
    sys.exit(1)
"

echo \"Mnemosyne health check passed\"
`} language="bash" />

Run this script via cron every 5 minutes. Replace thresholds with values appropriate for your deployment.
