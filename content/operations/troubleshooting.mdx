# Troubleshooting

Common issues and their resolutions.

## Installation Issues

### SQLite Version Too Old

<CodeBlock code={`# Check version
sqlite3 --version

# Upgrade (Ubuntu/Debian)
sudo apt-get update
sudo apt-get install sqlite3

# Or use conda
conda install -c conda-forge sqlite
`} language="bash" />

### FTS5 Not Available

<CodeBlock code={`# Verify FTS5
python -c "import sqlite3; conn = sqlite3.connect(':memory:'); conn.execute('CREATE VIRTUAL TABLE test USING fts5(content)')"

# If error, rebuild SQLite with FTS5
`} language="bash" />

## Runtime Issues

### Database Locked

<CodeBlock code={`# Enable WAL mode for better concurrency
sqlite3 mnemosyne.db "PRAGMA journal_mode=WAL;"

# Or in Python
mem = Mnemosyne(db_path=\"./mnemosyne.db\", session_id=\"diagnostic\")
`} language="bash" />

### Slow Queries

1. Enable WAL mode (see above)
2. Reduce Working Memory size
3. Verify SSD storage
4. Run consolidation to evict stale entries: `mem.sleep()`

### Memory Leaks

<CodeBlock code={`# Check memory stats
import psutil
process = psutil.Process()
print(f"RSS: {process.memory_info().rss / 1024 / 1024:.1f} MB")
`} language="python" />

## Data Issues

### Corrupted Database

<CodeBlock code={`# Check integrity
sqlite3 mnemosyne.db "PRAGMA integrity_check"

# If corrupted, restore from backup
cp mnemosyne-backup.db mnemosyne.db

# Or use disaster recovery
from mnemosyne.dr.recovery import restore_backup, verify_integrity
verify_integrity("mnemosyne.db")
restore_backup("mnemosyne-backup.db", "mnemosyne.db")
`} language="python" />

### Missing Embeddings

<CodeBlock code={`# Check memory stats (includes embedding counts)
hermes mnemosyne stats

# Or use the MCP diagnostic tool for full health report
mnemosyne_diagnose
`} language="bash" />

## Error Reference

| Error | Cause | Fix |
|-------|-------|-----|
| `sqlite3.OperationalError: database is locked` | Concurrent access | Enable WAL mode |
| `None` return from `mem.get()` | Wrong ID / entry not found | Verify ID format |
| `ValueError` / `KeyError` | Bad configuration or env var | Check `MNEMOSYNE_*` variables |

## Getting Help

1. Enable debug logging: `logging.getLogger("mnemosyne").setLevel(logging.DEBUG)`
2. Verify integrity: `sqlite3 mnemosyne.db "PRAGMA integrity_check"`
3. Check stats: `hermes mnemosyne stats`
4. Open an issue: https://github.com/AxDSan/mnemosyne/issues

<Callout type="info" title="Debug Mode">
Enable debug logging to see every SQL query and embedding request. This is verbose but invaluable for diagnosing issues.
</Callout>
