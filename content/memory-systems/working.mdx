# Working Memory

Working Memory is the active per-agent memory bank. Every call to `mnemosyne_remember()` stores one memory here. There is **no fixed capacity limit** — agents can call `remember()` as often as needed. The `sleep()` consolidation process handles growth by compressing old working memories into compact episodic summaries.

## How It Works

1. Agent calls `mnemosyne_remember(content="...")` → stored in working memory
2. Over time, working memory accumulates many individual entries
3. `mnemosyne_sleep()` triggers the consolidation cycle:
   - Old working memories are compressed into **episodic summaries**
   - These summaries persist indefinitely in a lower-priority tier
   - They still surface when queries match, just at reduced weight
4. Working memory stays responsive because old entries migrate out

<CodeBlock code={`# Store a working memory
mnemosyne_remember(
    content="User asked about vector search performance",
    importance=0.7
)

# Later: consolidate old entries
mnemosyne_sleep()
`} language="bash" />

## Consolidation with sleep()

The `sleep()` consolidation is the mechanism that keeps working memory from growing unbounded. Configure it via config.yaml:

```yaml
memory:
  mnemosyne:
    consolidation_threshold: 100   # Consolidate when working memory exceeds N entries
    consolidation_timer: 3600      # Seconds between consolidation runs
```

Or trigger it manually via the `mnemosyne_sleep` tool (also aliased as `hermes mnemosyne sleep` in Hermes).

## Episodic Summary Tiers

After consolidation, old working memories become episodic summaries in three tiers:

| Tier | Age | Priority | Description |
|------|-----|----------|-------------|
| **Active** | < 1 hour | High | Recent working memory entries, full detail |
| **Recent** | 1 hour — 7 days | Medium | Compressed summaries, still recallable |
| **Archive** | > 7 days | Low | Maximum compression, surfaces on strong match only |

<Callout type="info">
Three-tier degradation ensures no memory is ever deleted. Old memories gracefully fade into compact, lower-priority signals that still surface when the query really matters.
</Callout>

## Memory Banks

Working memory is scoped by [memory banks](/memory-systems/memory-banks). Each bank has its own independent working memory, so multi-agent setups can keep memories isolated.
