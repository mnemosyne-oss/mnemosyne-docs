# Installation

Mnemosyne is installed as a Python package and registered with your Hermes agent. This page walks through every step — from prerequisites to verification.

## Prerequisites

Before installing Mnemosyne, make sure you have:

- **Python 3.10 or higher** — check with `python --version`
- **Hermes Agent** already installed and initialized at `~/.hermes/hermes-agent/`
- **pip** or **uv** package manager available in your shell

## Quick Install

Three commands. That's all it takes.

**Step 1 — Activate the Hermes virtual environment:**

```bash
source ~/.hermes/hermes-agent/venv/bin/activate
```

**Step 2 — Install the package:**

Use the command that matches how your virtual environment was created:

```bash
# If your venv was created with python -m venv or virtualenv
pip install mnemosyne-memory[embeddings]

# If your venv was created with uv venv (pip is not included)
uv pip install mnemosyne-memory[embeddings]
```

**Step 3 — Register with Hermes:**

```bash
python -m mnemosyne.install
```

This creates the plugin symlink and updates `config.yaml` so Hermes knows about Mnemosyne.

## Install Options

### pip vs uv

Both work. `uv` is faster, but only if you use the matching install command:

| Venv created with | Install command |
|-------------------|-----------------|
| `python -m venv`  | `pip install mnemosyne-memory[embeddings]` |
| `virtualenv`      | `pip install mnemosyne-memory[embeddings]` |
| `uv venv`         | `uv pip install mnemosyne-memory[embeddings]` |

### Extras

Mnemosyne ships optional dependency groups you can install as needed:

| Extra | Command | What it adds |
|-------|---------|--------------|
| Core (not recommended) | `pip install mnemosyne-memory` | Base package with FTS5 keyword search only |
| **Embeddings** | `pip install mnemosyne-memory[embeddings]` | **+ fastembed for semantic vector search** |
| LLM consolidation | `pip install mnemosyne-memory[llm]` | + ctransformers for local LLM sleep consolidation |
| Everything | `pip install mnemosyne-memory[all]` | All optional dependencies |

## Optional: sqlite-vec for Performance

`sqlite-vec` is an **optional performance boost** for large datasets (100K+ memories). It moves vector similarity search into native C code inside SQLite. Without it, Mnemosyne uses an in-memory numpy fallback that works fine for most use cases.

```bash
pip install sqlite-vec
```

- **With `[embeddings]` only:** Full semantic search + hybrid retrieval via in-memory numpy cosine similarity. Works great for most deployments.
- **With `[embeddings]` + `sqlite-vec`:** Same semantic search, but vector queries run in native C inside SQLite. Faster on very large datasets.
- **Without `[embeddings]`:** FTS5 keyword search only, regardless of sqlite-vec.

Install sqlite-vec only if you need maximum performance on huge datasets.

## Verify Installation

Run the stats command through Hermes to confirm everything is wired up:

```bash
hermes mnemosyne stats
```

You should see a summary of memory tiers and database status. If the command is not found, the registration step may have failed — re-run `python -m mnemosyne.install`.

You can also verify the Python package directly:

```bash
python -c "import mnemosyne; print(mnemosyne.__version__)"
```

### Useful Hermes Commands

Once installed, these are the Mnemosyne commands available through Hermes:

| Command | Description |
|---------|-------------|
| `hermes mnemosyne stats` | Show memory statistics |
| `hermes mnemosyne version` | Show installed version |
| `hermes mnemosyne inspect [query]` | Search memories interactively |
| `hermes mnemosyne sleep` | Run consolidation (sleep cycle) |
| `hermes mnemosyne export --output <path>` | Export memories to JSON |
| `hermes mnemosyne import --input <path>` | Import memories from JSON |
| `hermes memory setup` | Interactive provider picker (curses UI) |
| `hermes memory status` | Show current memory provider config |

## Development Install (from Source)

If you're contributing or want to run the latest unreleased changes:

```bash
# Activate the Hermes venv first
source ~/.hermes/hermes-agent/venv/bin/activate

# Clone the repo
git clone https://github.com/axdsan/mnemosyne.git
cd mnemosyne

# Install in editable mode with dev dependencies
pip install -e ".[dev]"

# Register with Hermes
python -m mnemosyne.install
```

The `-e` flag installs in "editable" mode — changes to the source code take effect immediately without re-installing. Perfect for development and debugging.
