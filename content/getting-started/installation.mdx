# Installation

Mnemosyne v3.12.0 installs in one command. This is the only guide you need — every other page links here.

## For AI Agents

If you are an LLM agent installing Mnemosyne for a user, copy the block that matches the deployment target. Each block is self-contained and ends with a verify step.

**Standard Python SDK (any agent, includes semantic search):**

```bash
pip install mnemosyne-memory[embeddings]
```

**Hermes Agent plugin (most common — bundles core + embeddings automatically):**

```bash
pip install mnemosyne-hermes
```

**All features (embeddings + local LLM + MCP):**

```bash
pip install mnemosyne-memory[all]
```

**Verify after install:**

```bash
python -c "import mnemosyne; print(mnemosyne.__version__)"
```

For the detailed walkthrough (pipx, virtualenvs, MCP server setup, smoke test), see the sections below.

## Prerequisites

- **Python 3.10 or higher** — `python3 --version`
- **pipx** — `pip install pipx && pipx ensurepath`

## Install

```bash
pipx install mnemosyne-memory[all]
```

That's it. `[all]` gives you semantic vector search + local LLM consolidation. No database setup, no config files — Mnemosyne creates its SQLite database automatically on first use.

For speed:

```bash
export PIPX_DEFAULT_BACKEND=uv
pipx install mnemosyne-memory[all]
```

**Why pipx?** Isolated virtualenv that survives system rebuilds, clean uninstall, no conflicts with other Python projects.

## Verify

```bash
python3 -c "import mnemosyne; print(mnemosyne.__version__)"
# Should print: 3.12.0
```

## Upgrade

```bash
pipx upgrade mnemosyne-memory
```

## Extras

| Extra | What it adds | RAM |
|-------|-------------|-----|
| Bare install | FTS5 keyword search only. Good for Raspberry Pi or remote embedding API. | ~50 MB |
| `[embeddings]` | fastembed + sqlite-vec for semantic vector search | ~800 MB more |
| `[llm]` | ctransformers for local LLM sleep consolidation | ~600 MB more |
| `[all]` | Everything: embeddings + LLM consolidation. Recommended for 8 GB+ systems. | ~1.5 GB total |

**To use pipx with extras**: `pipx install 'mnemosyne-memory[embeddings]'` (quotes required around brackets).

**Hermes Agent users**: install `pipx install mnemosyne-hermes` instead — this bundles `[embeddings]` automatically. See full instructions on the [Hermes Plugin page](/api/hermes-plugin).

## Verify Installation

```bash
python3 -c "import mnemosyne; print(mnemosyne.__version__)"
```

Smoke test:

```python
from mnemosyne import Mnemosyne

mem = Mnemosyne()
mem.remember("Installation test", importance=0.1)
results = mem.recall("test")
print(f"Stored and retrieved {len(results)} memory")
```

## MCP Server Setup

Mnemosyne includes a built-in MCP server for Claude Desktop and other MCP clients:

```bash
# Add to Claude Desktop config (claude_desktop_config.json):
{
  "mcpServers": {
    "mnemosyne": {
      "command": "mnemosyne",
      "args": ["mcp"]
    }
  }
}
```

For SSE transport (web clients):

```bash
mnemosyne mcp --transport sse --port 8080
```

## Hermes Agent Integration

If you're using Mnemosyne as a Hermes plugin, see the [Hermes Plugin](/api/hermes-plugin) page for the complete setup guide including config and troubleshooting.

Quick commands:

```bash
pipx install mnemosyne-hermes
hermes config set memory.provider mnemosyne
hermes gateway restart
```

## Development Install (from Source)

```bash
git clone https://github.com/axdsan/mnemosyne.git
cd mnemosyne
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

The `-e` flag installs in "editable" mode — source changes take effect immediately.
