# LLM Backends (Host Adapter)

Mnemosyne can route LLM-backed operations — consolidation (`sleep()`) and structured fact extraction — through a **host-provided backend** instead of its own remote/local chain. This is essential when Mnemosyne runs inside **Hermes** and the user has an **OAuth-backed provider** (ChatGPT, Codex) that cannot be reached via `MNEMOSYNE_LLM_BASE_URL`.

## Why

The standalone LLM path expects an OpenAI-compatible URL plus API key:

```bash
MNEMOSYNE_LLM_BASE_URL=http://localhost:8080/v1
MNEMOSYNE_LLM_API_KEY=sk-...
```

OAuth/session-backed providers like `openai-codex` use provider-specific routing, headers, OAuth refresh, and Codex Responses API translation. Reproducing all of that inside Mnemosyne would be heavy. Reusing what Hermes already authenticates is dramatically simpler.

## How It Works

```
┌─────────────────┐     ┌──────────────────────┐     ┌─────────────────┐
│   Mnemosyne     │────▶│  Host LLM Backend    │────▶│  Hermes aux     │
│   sleep()       │     │  (registered at init) │     │  call_llm()     │
│   extract()     │     │                       │     │  (OAuth, Codex) │
└─────────────────┘     └──────────────────────┘     └─────────────────┘
```

The backend is a tiny **one-method interface**:

```python
class LLMBackend:
    name: str

    def complete(self, prompt, *, max_tokens, temperature, timeout,
                 provider=None, model=None) -> str | None:
        ...
```

## Configuration

```bash
# Required: opt in to the host backend
export MNEMOSYNE_HOST_LLM_ENABLED=true

# Optional overrides (Hermes auto-resolution otherwise)
export MNEMOSYNE_HOST_LLM_PROVIDER=openai-codex
export MNEMOSYNE_HOST_LLM_MODEL=gpt-5.1-mini
export MNEMOSYNE_HOST_LLM_N_CTX=32000
```

When `HOST_LLM_ENABLED` is unset or false, behavior is **identical to before** — remote URL → local GGUF → AAAK.

## Fallback Chain

When `MNEMOSYNE_HOST_LLM_ENABLED=true`:

```text
0. Host backend (if registered)
   ↓ on failure: skip remote URL entirely (A3 rule)
1. Local GGUF (ctransformers / llama-cpp-python)
   ↓ on failure
2. Return None / [] — caller falls back to AAAK encoding
```

When `MNEMOSYNE_HOST_LLM_ENABLED=false` or unset:

```text
0. Remote OpenAI-compatible API (if MNEMOSYNE_LLM_BASE_URL set)
   ↓ on failure
1. Local GGUF
   ↓ on failure
2. AAAK encoding
```

## The "Host-Skips-Remote" Rule (A3)

When the host attempt produces no usable text (errors, empty response, no extractable content), the **remote URL is skipped entirely**. The fallback goes straight to local GGUF.

This prevents Mnemosyne from accidentally routing memory content to a stale `MNEMOSYNE_LLM_BASE_URL` the user forgot to clear after switching to Hermes.

## For Other Agents

Any host can register its own backend:

```python
from mnemosyne.core.llm_backends import LLMBackend, set_host_llm_backend

class MyAgentBackend:
    name = "my-agent"

    def complete(self, prompt, *, max_tokens, temperature, timeout,
                 provider=None, model=None):
        # Route through your authenticated client
        return text_or_none

set_host_llm_backend(MyAgentBackend())
```

## Session Shutdown

Mnemosyne's `on_session_end()` hook runs sleep/consolidation in a **daemon thread** with a 15-second join timeout. If consolidation cannot finish in time (e.g., a slow host LLM call), the join returns and Hermes shutdown proceeds unblocked. The daemon thread continues in the background and is reaped when the process exits.

## Temperature Contracts

| Operation | Temperature | Reason |
|-----------|-------------|--------|
| **Fact extraction** | `0.0` | Deterministic — avoids near-duplicate facts on re-ingest |
| **Consolidation** | `0.3` | Paraphrasing variance is acceptable |
