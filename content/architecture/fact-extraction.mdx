# Fact Extraction

Mnemosyne v3 can extract structured facts from memory content using an LLM, storing them as triples for knowledge-graph queries.

## Fallback Chain

1. **Host LLM backend** (Hermes auxiliary client) — when `MNEMOSYNE_HOST_LLM_ENABLED=true` and running inside Hermes. Uses authenticated provider including OAuth-backed subscriptions like ChatGPT/Codex.
2. **Remote OpenAI-compatible API** — fastest, best quality
3. **Local ctransformers GGUF** — offline, no API key
4. **Skip** — graceful degradation, never raises

<Callout type="info">
Fact extraction is **opt-in**. Pass `extract=True` to `remember()`.
</Callout>

<Callout type="warning" title="Determinism">
Fact extraction uses `temperature=0.0` for deterministic output. Re-ingesting the same content produces identical facts, avoiding near-duplicate writes to the facts table.
</Callout>

## Usage

<CodeBlock code={`from mnemosyne import Mnemosyne

mem = Mnemosyne()
mid = mem.remember(
    "Mnemosyne uses SQLite for storage and fastembed for embeddings",
    importance=0.8,
    extract=True
)
`} language="python" />

## Standalone Usage

<CodeBlock code={`from mnemosyne.core.extraction import extract_facts, extract_facts_safe

facts = extract_facts("The project deadline is June 15th.")
facts = extract_facts_safe("Always safe, never raises.")
`} language="python" />

## Configuration

| Variable | Description |
|----------|-------------|
| `MNEMOSYNE_LLM_BASE_URL` | OpenAI-compatible API endpoint |
| `MNEMOSYNE_LLM_API_KEY` | API key |
| `MNEMOSYNE_LLM_MODEL` | Model name |
