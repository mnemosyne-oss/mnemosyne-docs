# Cron Deployment

Run Mnemosyne consolidation and maintenance as scheduled batch processes.

## Use Cases

- Daily consolidation jobs
- Periodic backups via disaster recovery module
- Database maintenance

## Crontab Example

<CodeBlock code={`# Edit crontab
crontab -e

# Daily consolidation at 3 AM via Hermes
0 3 * * * cd /path/to/project && hermes mnemosyne sleep >> /var/log/mnemosyne/consolidate.log 2>&1

# Daily backup at 4 AM
0 4 * * * /opt/mnemosyne/venv/bin/mnemosyne backup --output-dir /backups >> /var/log/mnemosyne/backup.log 2>&1
`} language="bash" />

## Consolidation via Hermes

Consolidation runs through the Hermes plugin command:

<CodeBlock code={`# Run consolidation manually
hermes mnemosyne sleep

# Or programmatically in Python
from mnemosyne import Mnemosyne

mem = Mnemosyne(db_path="/data/mnemosyne.db")
mem.sleep()  # Run consolidation
`} language="python" />

<Callout type="warning" title="No Standalone Consolidate Command">
There is no `mnemosyne-consolidate` CLI tool. Consolidation is only available via `mnemosyne_sleep` tool or by calling `mem.sleep()` programmatically.
</Callout>

## Backup

Backup via the CLI:

<CodeBlock code={`# Automated backup via CLI
mnemosyne backup --output-dir /backups

# Programmatically
from pathlib import Path
from mnemosyne.dr.recovery import create_backup

backup_path = create_backup(
    db_path=Path("/data/mnemosyne.db"),
    backup_dir=Path("/backups")
)
`} language="python" />

## Monitoring Cron Jobs

<CodeBlock code={`# Add to crontab
MAILTO=admin@example.com

# Log to file
0 3 * * * cd /path/to/project && hermes mnemosyne sleep >> /var/log/mnemosyne/consolidate.log 2>&1
`} language="bash" />

<Callout type="success" title="Cron Best Practice">
Use absolute paths for all commands and files. Cron runs with a minimal PATH. Test scripts manually before adding to crontab.
</Callout>
