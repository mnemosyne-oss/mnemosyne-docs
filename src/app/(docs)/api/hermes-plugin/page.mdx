# Hermes Plugin

First-class integration with the Hermes agent framework.

## Install Profile Comparison

| Profile | When to use | RAM | Key tradeoff |
|---------|-------------|-----|-------------|
| `mnemosyne-memory` (core) | Low-resource (Raspberry Pi, 1 GB VPS), or when using a remote embedding API | ~50 MB | No local embeddings. Point `MNEMOSYNE_EMBEDDING_API_URL` to an external endpoint. |
| `mnemosyne-memory[embeddings]` | Mid-range systems with local embedding support | ~800 MB | Adds `fastembed` for local vector generation. Best for single-user desktop agents. |
| `mnemosyne-memory[all]` | Full-featured — local embeddings + local LLM consolidation | ~1.5 GB | Adds `sentence-transformers` + local LLM deps. Maximum capability. Recommended for 8 GB+ RAM systems. |
| `mnemosyne-hermes` | Hermes Agent users — always pair with one of the above | Same as base | Wraps core library with plugin manifest + entry points. Run `hermes config set memory.provider mnemosyne` after install. |

**Hardware guidance:** Core alone runs on a Raspberry Pi 4 (4 GB) with ~300 MB free for LLM. `[embeddings]` needs at least 2 GB free RAM. `[all]` recommends 8 GB+.

## Installation

**Do NOT install inside Hermes' managed venv** — `hermes update` rebuilds
the venv and wipes extra packages. Use pipx (recommended) or a dedicated
virtualenv.

<CodeBlock code={`# Recommended: survives Hermes updates
pipx install mnemosyne-hermes
hermes config set memory.provider mnemosyne
hermes gateway restart

# For speed with pipx
export PIPX_DEFAULT_BACKEND=uv
pipx install mnemosyne-hermes

# Upgrade
pipx upgrade mnemosyne-hermes
hermes gateway restart
`} language="bash" />

### Fallback: standalone venv

<CodeBlock code={`python3 -m venv ~/.hermes/mnemosyne-venv
~/.hermes/mnemosyne-venv/bin/pip install mnemosyne-hermes
~/.hermes/mnemosyne-venv/bin/mnemosyne-hermes install --force
hermes config set memory.provider mnemosyne
hermes gateway restart
`} language="bash" />

### Disable legacy memory

When Mnemosyne is active, disable Hermes' built-in MEMORY.md/USER.md storage
to avoid duplication and token waste.

**DO NOT run `hermes tools disable memory`** — this kills Mnemosyne's tool
surface along with the built-in memory. The `"memory"` toolset key gates
both the built-in tool AND memory provider tools (source: `agent/agent_init.py:1068-1079`).

Instead, add to `~/.hermes/config.yaml`:

<CodeBlock code={`memory:
  memory_enabled: false
  user_profile_enabled: false
  provider: mnemosyne
`} language="yaml" />

The `memory_enabled: false` + `user_profile_enabled: false` disables
the built-in file-based memory store while keeping the `"memory"` toolset
active so Mnemosyne's tools surface correctly.

When enabled, Mnemosyne automatically injects relevant memories into agent context:

## Plugin Hooks

Mnemosyne registers the following Hermes hooks:

| Hook | Timing | Action |
|------|--------|--------|
| `on_turn_start` | Start of each agent turn | Prefetch identity + inject relevant memories |
| `on_session_end` | Session close | Trigger consolidation, store conversation summary |
| `on_memory_write` | After Hermes builtin memory writes | Sync to Mnemosyne for centralized recall |

## Tool Schema

Mnemosyne exposes **25 tools** to agents through the Hermes plugin. See the complete [Tool Schema Reference](/api/tool-schema) for every tool, parameter, and default value.

Key tools:

| Tool | Purpose |
|------|---------|
| `mnemosyne_remember` | Store memories with entity/fact extraction |
| `mnemosyne_recall` | Hybrid vector+FTS5 search with configurable weights |
| `mnemosyne_sleep` | Run consolidation — compress working memory into episodic |
| `mnemosyne_stats` | Memory system statistics (counts, BEAM tiers, DB size) |
| `mnemosyne_forget` | Permanently delete a memory by ID |
| `mnemosyne_update` | Update content or importance of an existing memory |
| `mnemosyne_get` | Retrieve a single memory by ID |
| `mnemosyne_validate` | Collaborative ownership (attest, update, invalidate) |
| `mnemosyne_invalidate` | Mark a memory as expired/superseded |
| `mnemosyne_export` / `mnemosyne_import` | Backup and migration |
| `mnemosyne_diagnose` | PII-safe diagnostics (deps, DB state, vector readiness) |
| `mnemosyne_triple_add` / `mnemosyne_triple_query` | Knowledge graph operations |
| `mnemosyne_graph_link` / `mnemosyne_graph_query` | Semantic graph traversal |
| `mnemosyne_scratchpad_*` | Temporary workspace (write, read, clear) |
| `mnemosyne_shared_*` | Cross-agent surface memory (remember, recall, forget, stats) |
| `mnemosyne_remember_canonical` / `mnemosyne_recall_canonical` | Owner-scoped canonical facts (v3.7.0) |

All tools are documented with full parameter schemas on the [Tool Schema](/api/tool-schema) page.

## Troubleshooting

### Plugin not loading

<CodeBlock code={`hermes plugins list
# Verify mnemosyne appears

hermes mnemosyne stats
# Check memory statistics
`} language="bash" />

### Context not injected

Verify `auto_context` is enabled and memories exist:

<CodeBlock code={`hermes plugins list
hermes mnemosyne stats
`} language="bash" />
