# API Overview

Mnemosyne exposes multiple interfaces: Python SDK, Hermes plugin tools, MCP server, and more.

## Interface Comparison

| Interface | Best For | Latency | Setup |
|-----------|----------|---------|-------|
| Python SDK | Direct integration, scripts | Lowest | `pip install` |
| Hermes Plugin | Agent frameworks | Low | Plugin config |
| MCP Server | Claude Desktop, LLM clients | Low | `mnemosyne mcp` |

## Python SDK

The primary interface for Python applications:

<CodeBlock code={`from mnemosyne import Mnemosyne

mem = Mnemosyne()
mem.remember("User prefers dark mode.")
results = mem.recall("user preferences")
`} language="python" />

## Hermes Plugin

First-class integration with the Hermes agent framework:

<CodeBlock code={`# hermes.yaml
plugins:
  - name: mnemosyne-memory
    source: pip
    config:
      db_path: ./memory.db
      embedding_model: text-embedding-3-small
      auto_context: true
      context_injection:
        enabled: true
        max_memories: 5
        min_relevance: 0.7
`} language="yaml" />

## MCP Server

Mnemosyne ships a built-in Model Context Protocol (MCP) server for integration with Claude Desktop and other MCP-compatible clients:

<CodeBlock code={`# stdio transport (Claude Desktop, etc.)
mnemosyne mcp

# SSE transport (web clients)
mnemosyne mcp --transport sse --port 8080

# Scoped to a specific memory bank
mnemosyne mcp --bank project_a
`} language="bash" />

The MCP server exposes 23 tools including `mnemosyne_remember`, `mnemosyne_recall`, `mnemosyne_sleep`, `mnemosyne_scratchpad_read`, `mnemosyne_scratchpad_write`, `mnemosyne_stats`, `mnemosyne_get`, `mnemosyne_forget`, `mnemosyne_shared_remember`, `mnemosyne_shared_recall`, `mnemosyne_validate`, `mnemosyne_graph_link`, and more.

<Callout type="info" title="SDK Version Compatibility">
Python SDK, Hermes plugin, and MCP server share the same core version. Ensure all components use matching versions for full compatibility.
</Callout>
