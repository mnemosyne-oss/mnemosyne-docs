# API Overview

Mnemosyne exposes multiple interfaces: Python SDK, Hermes plugin tools, and a legacy REST API.

## Interface Comparison

| Interface | Best For | Latency | Setup |
|-----------|----------|---------|-------|
| Python SDK | Direct integration, scripts | Lowest | `pip install` |
| Hermes Plugin | Agent frameworks | Low | Plugin config |
| REST API (Legacy) | Remote services, microservices | Medium | `python -m mnemosyne` |

## Python SDK

The primary interface for Python applications:

<CodeBlock code={`from mnemosyne import Memory

mem = Memory()
mem.remember("User prefers dark mode.")
results = mem.recall("user preferences")
`} language="python" />

## Hermes Plugin

First-class integration with the Hermes agent framework:

<CodeBlock code={`# hermes.yaml
plugins:
  - name: mnemosyne-memory
    source: pip
    config:
      db_path: ./memory.db
      embedding_model: text-embedding-3-small
      auto_context: true
      context_injection:
        enabled: true
        max_memories: 5
        min_relevance: 0.7
`} language="yaml" />

## REST API (Legacy)

The legacy FastAPI server embedded in `mnemosyne/cli.py`:

<CodeBlock code={`python -m mnemosyne --port 8090
`} language="bash" />

Example request:

<CodeBlock code={`curl -X POST http://localhost:8090/store \
  -H "Content-Type: application/json" \
  -d '{"content": "User prefers dark mode.", "tags": ["preferences"]}'
`} language="bash" />

<Callout type="info" title="SDK Version Compatibility">
Python SDK, Hermes plugin, and REST API share the same core version. Ensure all components use matching versions for full compatibility.
</Callout>
