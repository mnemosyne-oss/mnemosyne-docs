## Tool Selection Guide

<Callout type="info" title="MCP vs Hermes Plugin Tools">
**All 23 tools** are available via the Hermes Agent plugin. The MCP server exposes a subset (remember, recall, sleep, scratchpad, stats) plus triple tools by name but not by discovery.

> Full tool list: `mnemosyne_remember`, `mnemosyne_recall`, `mnemosyne_shared_remember`, `mnemosyne_shared_recall`, `mnemosyne_shared_forget`, `mnemosyne_shared_stats`, `mnemosyne_sleep`, `mnemosyne_stats`, `mnemosyne_get`, `mnemosyne_forget`, `mnemosyne_update`, `mnemosyne_invalidate`, `mnemosyne_validate`, `mnemosyne_triple_add`, `mnemosyne_triple_query`, `mnemosyne_scratchpad_write`, `mnemosyne_scratchpad_read`, `mnemosyne_scratchpad_clear`, `mnemosyne_export`, `mnemosyne_import`, `mnemosyne_diagnose`, `mnemosyne_graph_query`, `mnemosyne_graph_link`.
</Callout>

| Goal | Tool | Example |
|------|------|---------|
| Store general info | `mnemosyne_remember` | User preferences, decisions |
| Retrieve context | `mnemosyne_recall` | "What did we decide?" |
| Store cross-agent surface info | `mnemosyne_shared_remember` | User metadata, workflow state |
| Search shared surface | `mnemosyne_shared_recall` | Cross-agent preferences |
| Delete shared surface memory | `mnemosyne_shared_forget` | Remove stale shared state |
| Check shared surface health | `mnemosyne_shared_stats` | Shared DB path, counts |
| Consolidate memories | `mnemosyne_sleep` | End-of-session cleanup |
| Check system health | `mnemosyne_stats` | Memory counts, DB size |
| Run diagnostics | `mnemosyne_diagnose` | DB integrity, index health |
| Remove bad data | `mnemosyne_invalidate` | Outdated information |
| Permanently delete memory | `mnemosyne_forget` | Wrong information, retractions |
| Update existing memory | `mnemosyne_update` | Correct facts, promote inferences |
| Retrieve single memory by ID | `mnemosyne_get` | "Show me memory abc-123" |
| Cross-agent validation | `mnemosyne_validate` | Attest or correct memory |
| Store relational facts | `mnemosyne_triple_add` | Team structure, dependencies |
| Precise lookup | `mnemosyne_triple_query` | "Who reports to Carol?" |
| Export memories | `mnemosyne_export` | Backup, migration |
| Import memories | `mnemosyne_import` | Restore from backup |
| Write scratchpad | `mnemosyne_scratchpad_write` | Chain-of-thought planning |
| Read scratchpad | `mnemosyne_scratchpad_read` | Review planning notes |
| Clear scratchpad | `mnemosyne_scratchpad_clear` | Reset workspace |
| Graph traversal | `mnemosyne_graph_query` | Find related memories |
| Link memories | `mnemosyne_graph_link` | Declare semantic edges |

<Callout type="success" title="Schema Validation">
Tool calls are validated against these schemas before execution. Invalid parameters return a structured error that the agent can use to self-correct.
</Callout>
