# BEAM Architecture Overview

BEAM (Biological-inspired Episodic-Associative Memory) is the core architecture powering Mnemosyne. It models human memory organization with four distinct tiers, each optimized for different retention and retrieval patterns.

### Working Memory

The agent's immediate context window. Holds recent observations, conversation history, and active tool results. Fastest access (&lt;100ms), limited capacity (~1000 entries), and subject to eviction.

**Key characteristics:**
- **Latency**: Sub-100ms median
- **Capacity**: Configurable (default 1000 entries)
- **Eviction**: LRU with importance weighting
- **Persistence**: Survives process restarts via SQLite

### Episodic Memory

Long-term storage for experiences, conversations, and events. Organized chronologically with rich metadata. Uses hybrid retrieval combining vector similarity and full-text search.

**Key characteristics:**
- **Retention**: Permanent (until explicit deletion)
- **Retrieval**: Hybrid (vector + FTS5)
- **Compression**: AAAK algorithm for context windows
- **Consolidation**: Automatic during sleep cycles

### Semantic Memory

Structured knowledge extracted from experiences. Stores facts, relationships, and concepts as a temporal knowledge graph with subject-predicate-object triples.

**Key characteristics:**
- **Format**: Temporal knowledge graph (triples)
- **Query**: Graph traversal + vector similarity
- **Update**: Incremental fact extraction from Working Memory
- **Schema**: Flexible, no predefined ontology

### Scratchpad

A temporary workspace for reasoning, planning, and intermediate computations. Analogous to human working memory's manipulation buffer.

**Key characteristics:**
- **Lifetime**: Session-bound (cleared on agent restart)
- **Use case**: Chain-of-thought, planning, scratch calculations
- **Integration**: Flushes insights to Working Memory
- **Isolation**: Per-agent, not shared

## Sleep Consolidation

Mnemosyne runs background "sleep" cycles that:

1. **Compress** Working Memory entries using AAAK algorithm
2. **Promote** high-importance entries to Episodic Memory
3. **Extract** facts and relationships to Semantic Memory
4. **Evict** low-importance entries per retention policy

Configure interval via `MNEMOSYNE_CONSOLIDATION_INTERVAL` (default: 3600s).

## Design Philosophy

Mnemosyne's architecture prioritizes:

1. **Locality**: All data in SQLite — no network calls for retrieval
2. **Transparency**: Every operation is inspectable and debuggable
3. **Efficiency**: Sub-100ms retrieval for typical workloads
4. **Flexibility**: Works standalone or as a Hermes plugin
5. **Privacy**: No data leaves your machine by default

<Callout type="info" title="Comparison with Vector-Only Stores">
Unlike pure vector databases (Pinecone, Weaviate), Mnemosyne combines dense embeddings with structured storage, full-text search, and temporal relationships. This hybrid approach delivers more precise retrieval for agent use cases where exact facts matter as much as semantic similarity.
</Callout>
