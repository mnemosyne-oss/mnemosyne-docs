# Entity Extraction

Mnemosyne v2 introduces lightweight entity extraction using regex patterns and pure-Python Levenshtein fuzzy matching. No NLP dependencies required.

## How It Works

Entity extraction identifies proper nouns, handles, hashtags, and quoted phrases in memory content, storing them as triples for fuzzy entity-aware recall.

<Callout type="info">
Entity extraction is **opt-in**. Pass `extract_entities=True` to `remember()`. It adds ~0.01ms overhead per extraction.
</Callout>

## Extraction Patterns

| Pattern | Example |
|---------|---------|
| @mentions | `@abdias` |
| #hashtags | `#python` |
| Quoted phrases | `"Project Alpha"` |
| Capitalized sequences (2-5 words) | `New York`, `Abdias J` |
| Single capitalized words | `Mnemosyne` |

## Usage

<CodeBlock code={`from mnemosyne import Mnemosyne

mem = Mnemosyne()
mid = mem.remember(
    "Abdias founded Mnemosyne in New York",
    importance=0.9,
    extract_entities=True
)

# Entities stored as triples:
# (mid, "mentions", "Abdias")
# (mid, "mentions", "Mnemosyne")
# (mid, "mentions", "New York")
`} language="python" />

## Fuzzy Matching

<CodeBlock code={`from mnemosyne.core.entities import find_similar_entities

matches = find_similar_entities(
    "Abdias",
    ["Abdias J", "Python", "New York"],
    threshold=0.7
)
# Returns: [("Abdias J", 0.925)]
`} language="python" />

## Trade-offs

| Aspect | Mnemosyne | Full NLP (spaCy) |
|--------|-----------|-------------------|
| Speed | ~0.01ms per extraction | Heavier |
| Dependencies | Zero | spaCy + model (~50MB) |
| Accuracy | Good for proper nouns | Higher recall |

<Callout type="warning">
Misses pronouns, complex references, and co-reference resolution. For production NER, integrate spaCy externally.
</Callout>
