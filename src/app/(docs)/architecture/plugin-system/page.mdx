# Plugin System

Extensible plugin architecture for custom memory behavior.

## Creating a Plugin

<CodeBlock code={`from mnemosyne.core.plugins import MnemosynePlugin

class AuditPlugin(MnemosynePlugin):
    name = "audit-logger"
    
    def on_remember(self, memory):
        print(f"[AUDIT] Stored: {memory.id}")
    
    def on_recall(self, memory):
        print(f"[AUDIT] Recalled: {memory.get('id')}")
    
    def on_consolidate(self, summary):
        print(f"[AUDIT] Consolidated: {summary.get('memory_count', 0)}")
    
    def on_invalidate(self, memory_id):
        print(f"[AUDIT] Invalidated: {memory_id}")
`} language="python" />

## Registering

<CodeBlock code={`mem = Mnemosyne()

mem.plugins.register_plugin("audit-logger", AuditPlugin)
mem.plugins.discover_plugins()
plugins = mem.plugins.list_plugins()
mem.plugins.unload_plugin("audit-logger")
`} language="python" />

## Built-in Plugins

| Plugin | Description |
|--------|-------------|
| LoggingPlugin | Logs all operations |
| MetricsPlugin | Tracks counts and latencies |
| FilterPlugin | Filters by custom criteria |

## Auto-Discovery

Drop Python files into `~/.hermes/mnemosyne/plugins/`. Any `MnemosynePlugin` subclass is auto-loaded.
