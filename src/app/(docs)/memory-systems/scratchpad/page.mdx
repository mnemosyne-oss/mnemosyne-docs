# Scratchpad Memory

Ephemeral scratchpad for transient, in-session notes. Simple write/read/clear pattern.

## How It Works

The scratchpad stores a single mutable string per bank. Calling `scratchpad_write` replaces it. No history — it's a sticky note, not a log.

## API

| Method | Description |
|--------|-------------|
| `scratchpad_write(content=...)` | Replace the scratchpad string |
| `scratchpad_read()` | Get the current scratchpad string |
| `scratchpad_clear()` | Empty the scratchpad |

<CodeBlock code={`# Write a note
mnemosyne_scratchpad_write(content="User's name is Alice, works at Acme Corp")

# Read it back
mnemosyne_scratchpad_read()
# Returns: "User's name is Alice, works at Acme Corp"

# Clear when done
mnemosyne_scratchpad_clear()
`} language="bash" />

## Use Cases

- **Session context**: Agent keeps ephemeral user info (name, preferences, current task)
- **Temporary state**: Multi-step calculation results that don't need permanent memory
- **Debugging**: Dump intermediate state during development

<Callout type="info">
Scratchpad is deliberately simple. For structured temporary state, use [Working Memory](/memory-systems/working). For permanent memory, use [remember](/memory-systems/working).
</Callout>
