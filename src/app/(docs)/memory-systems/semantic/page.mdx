# Semantic Memory

Semantic Memory stores structured relationships between entities as subject-predicate-object triples with temporal validity tracking. It's managed by the separate `TripleStore` class.

## How It Works

Each triple has:
- **Subject**: The entity being described (e.g., "user")
- **Predicate**: The relationship (e.g., "prefers")
- **Object**: The value (e.g., "neovim")
- **valid_from** (optional): When this fact became true
- **valid_until** (optional): When it stopped being true (for historical facts)

Facts are stored in tables indexed by subject+predicate for fast lookup. Temporal validity means you can query "what did Alice prefer last month?" and get correct historical answers.

## Triple CRUD

<CodeBlock code={`# Store a fact
mnemosyne_triple_add(
    subject="alice",
    predicate="prefers",
    object="neovim"
)

# Set temporal range
mnemosyne_triple_add(
    subject="bob",
    predicate="role",
    object="engineer",
    valid_from="2025-06-01"
)

# End a fact (it stays in history)
mnemosyne_triple_end(
    subject="bob",
    predicate="role",
    object="engineer"
)
`} language="bash" />

## Querying

<CodeBlock code={`# All facts about Alice
mnemosyne_triple_query(subject="alice")

# What does Alice prefer?
mnemosyne_triple_query(subject="alice", predicate="prefers")

# Who prefers Neovim?
mnemosyne_triple_query(predicate="prefers", object="neovim")

# As of a specific date
mnemosyne_triple_query(
    subject="bob",
    predicate="role",
    valid_at="2025-07-15"
)
`} language="bash" />

## When to Use Semantic vs. Episodic

| | Semantic Memory | Episodic Memory |
|---|---|---|
| **What it stores** | Explicit facts (S→P→O) | Implicit learnings from conversations |
| **How it's queried** | Direct lookup by subject/predicate | Semantic vector search |
| **Temporal precision** | Exact validity windows | Approximate (recency weighting) |
| **Best for** | Known facts, preferences, roles | Inferred learnings, conversation context |

<Callout type="info">
Semantic Memory is complementary to Episodic Memory. Use triples for things you know explicitly (Alice's preferences, Bob's role, company policies). Use episodic/implicit memory for things you learned from context but can't name precisely.
</Callout>

## Graph Search

The semantic knowledge graph can be traversed with graph tools:

<CodeBlock code={`# Link two memories with a semantic edge
mnemosyne_graph_link(
    source_id="mem_abc123",
    target_id="mem_def456",
    relationship="references",
    weight=0.8
)

# Traverse from a seed memory
mnemosyne_graph_query(
    seed_memory_id="mem_abc123",
    max_hops=2,
    min_weight=0.5
)
`} language="bash" />
