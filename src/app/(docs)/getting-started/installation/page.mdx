# Installation

Mnemosyne is installed as a Python package and registered with your Hermes agent. This page walks through every step — from prerequisites to verification.

## Prerequisites

Before installing Mnemosyne, make sure you have:

- **Python 3.10 or higher** — check with `python --version`
- **Hermes Agent** already installed and initialized at `~/.hermes/hermes-agent/`
- **pip** or **uv** package manager available in your shell

<Callout type="info" title="Don't have Hermes yet?">
If you haven't set up Hermes Agent, do that first. Mnemosyne is a plugin that runs *inside* the Hermes ecosystem — it won't work as a standalone tool without it.
</Callout>

## Quick Install

Three commands. That's all it takes.

**Step 1 — Activate the Hermes virtual environment:**

<CodeBlock code={`source ~/.hermes/hermes-agent/venv/bin/activate`} language="bash" />

<Callout type="warning" title="Required step">
You *must* activate the venv before running pip. Without it, the package installs into the wrong Python environment and Hermes won't find it.
</Callout>

**Step 2 — Install the package:**

Then install Mnemosyne **with full semantic search support**:

<CodeBlock code={`# If your venv was created with python -m venv or virtualenv
pip install mnemosyne-memory[embeddings] sqlite-vec

# If your venv was created with uv venv (pip is not included)
uv pip install mnemosyne-memory[embeddings] sqlite-vec`} language="bash" />

<Callout type="warning" title="uv venv does not include pip">
If you created the Hermes venv with `uv venv`, running `pip install` will fail with "command not found." This is by design — uv replaces pip with `uv pip`. Always use `uv pip install` in uv-created environments.
</Callout>

<Callout type="danger" title="Both fastembed AND sqlite-vec are required for semantic search">
To unlock semantic vector search and hybrid retrieval, you need **both** dependencies:

1. **`[embeddings]`** — installs fastembed, which converts text into semantic vectors
2. **`sqlite-vec`** — stores those vectors in SQLite for efficient similarity search

Without sqlite-vec, fastembed generates vectors but they are discarded — you still get only FTS5 keyword search. Without `[embeddings]`, there are no vectors at all. Both are required for:

- Semantic similarity search
- Context-aware retrieval
- Meaning-based memory matching
- Hybrid search (vector + text combined)
</Callout>

**Step 3 — Register with Hermes:**

<CodeBlock code={`python -m mnemosyne.install`} language="bash" />

This creates the plugin symlink and updates `config.yaml` so Hermes knows about Mnemosyne.

## Install Options

### pip vs uv

Both work. `uv` is faster, but only if you use the matching install command:

| Venv created with | Install command |
|-------------------|-----------------|
| `python -m venv`  | `pip install mnemosyne-memory[embeddings] sqlite-vec` |
| `virtualenv`      | `pip install mnemosyne-memory[embeddings] sqlite-vec` |
| `uv venv`         | `uv pip install mnemosyne-memory[embeddings] sqlite-vec` |

<CodeBlock code={`# Standard pip (venv created with python -m venv)
pip install mnemosyne-memory[embeddings] sqlite-vec

# uv (venv created with uv venv)
uv pip install mnemosyne-memory[embeddings] sqlite-vec`} language="bash" />

### Extras

Mnemosyne ships optional dependency groups you can install as needed:

| Extra | Command | What it adds |
|-------|---------|--------------|
| Core (not recommended) | `pip install mnemosyne-memory` | Base package with FTS5 keyword search only |
| **Semantic Search** | `pip install mnemosyne-memory[embeddings] sqlite-vec` | **Full semantic + hybrid search** |
| LLM consolidation | `pip install mnemosyne-memory[llm]` | + ctransformers for local LLM sleep consolidation |
| Everything | `pip install mnemosyne-memory[all]` | All optional dependencies |

<Callout type="danger">
**You need both `[embeddings]` AND `sqlite-vec`** for semantic search to actually work:

- **`[embeddings]`** installs fastembed (converts text to vectors)
- **`sqlite-vec`** stores vectors in SQLite for similarity search

Without sqlite-vec, fastembed generates vectors but they are discarded. Without `[embeddings]`, there are no vectors at all. Install both together.
</Callout>

## Verify Installation

Run the stats command through Hermes to confirm everything is wired up:

<CodeBlock code={`hermes mnemosyne stats`} language="bash" />

You should see a summary of memory tiers and database status. If the command is not found, the registration step may have failed — re-run `python -m mnemosyne.install`.

You can also verify the Python package directly:

<CodeBlock code={`python -c "import mnemosyne; print(mnemosyne.__version__)"`} language="bash" />

### Useful Hermes Commands

Once installed, these are the Mnemosyne commands available through Hermes:

| Command | Description |
|---------|-------------|
| `hermes mnemosyne stats` | Show memory statistics |
| `hermes mnemosyne version` | Show installed version |
| `hermes mnemosyne inspect [query]` | Search memories interactively |
| `hermes mnemosyne sleep` | Run consolidation (sleep cycle) |
| `hermes mnemosyne export --output <path>` | Export memories to JSON |
| `hermes mnemosyne import --input <path>` | Import memories from JSON |
| `hermes memory setup` | Interactive provider picker (curses UI) |
| `hermes memory status` | Show current memory provider config |

<Callout type="danger" title="Common mistake">
Do **not** use `hermes plugin install mnemosyne` — that command doesn't exist. The correct Hermes plugin commands are `hermes plugins install <git-url>` (note the plural *plugins*), which expects a Git repository URL. Mnemosyne is installed via `pip` and registered with `python -m mnemosyne.install`, not through the plugin manager.
</Callout>

## Optional: sqlite-vec for Vector Performance

Mnemosyne uses **fastembed** (installed via `[embeddings]`) to generate semantic vectors. **sqlite-vec** stores those vectors in SQLite for efficient similarity search. Both are required for semantic + hybrid retrieval.

If you already installed both, no further action needed. If you only installed `[embeddings]` and are getting keyword-only results, add sqlite-vec:

<CodeBlock code={`pip install sqlite-vec`} language="bash" />

<Callout type="info" title="sqlite-vec is required, not optional, for semantic search">
- **With both `[embeddings]` + `sqlite-vec`:** Full semantic search + hybrid retrieval (vector + FTS5 combined)
- **With `[embeddings]` only:** fastembed generates vectors but they are discarded — you still get only FTS5 keyword search
- **With neither:** FTS5 keyword search only

Both are required for semantic similarity, context-aware retrieval, and hybrid search.
</Callout>

## Development Install (from Source)

If you're contributing or want to run the latest unreleased changes:

<CodeBlock code={`# Activate the Hermes venv first
source ~/.hermes/hermes-agent/venv/bin/activate

# Clone the repo
git clone https://github.com/axdsan/mnemosyne.git
cd mnemosyne

# Install in editable mode with dev dependencies
pip install -e ".[dev]"

# Register with Hermes
python -m mnemosyne.install`} language="bash" />

<Callout type="info" title="Editable mode">
The `-e` flag installs in "editable" mode — changes to the source code take effect immediately without re-installing. Perfect for development and debugging.
</Callout>
