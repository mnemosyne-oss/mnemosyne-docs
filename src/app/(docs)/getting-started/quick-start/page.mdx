# Quick Start

Get Mnemosyne running in under 5 minutes. This guide covers installation, basic configuration, and your first memory operations.

## Prerequisites

- Python 3.10 or higher
- pip or uv package manager

## Install

First, activate the Hermes agent virtual environment:

<CodeBlock code={`source ~/.hermes/hermes-agent/venv/bin/activate`} language="bash" />

Then install Mnemosyne **with embeddings support**:

<CodeBlock code={`# If your venv was created with python -m venv or virtualenv
pip install mnemosyne-memory[embeddings]

# If your venv was created with uv venv (pip is not included)
uv pip install mnemosyne-memory[embeddings]`} language="bash" />

<Callout type="warning" title="uv venv does not include pip">
If you created the Hermes venv with `uv venv`, running `pip install` will fail with "command not found." This is by design — uv replaces pip with `uv pip`. Always use `uv pip install` in uv-created environments.
</Callout>

<Callout type="danger" title="fastembed + sqlite-vec are both needed for semantic search">
To get semantic vector search and hybrid retrieval, you need **both**:

1. **`[embeddings]`** — installs fastembed, which converts text into semantic vectors
2. **`sqlite-vec`** — stores those vectors in SQLite for efficient similarity search

Install both:

<CodeBlock code={`pip install mnemosyne-memory[embeddings] sqlite-vec`} language="bash" />

Without sqlite-vec, fastembed generates vectors but they are discarded — you still get FTS5 keyword search only. Without `[embeddings]`, there are no vectors at all. Both are required for:

- Semantic similarity search
- Context-aware retrieval
- Meaning-based memory matching
- Hybrid search (vector + text combined)
</Callout>

Or with uv (recommended for speed):

<CodeBlock code={`uv pip install mnemosyne-memory[embeddings]`} language="bash" />

## Initialize the Database

Run the built-in installer to set up the SQLite database and schema:

<CodeBlock code={`python -m mnemosyne.install`} language="bash" />

> **Note:** Mnemosyne uses SQLite with FTS5 for full-text search by default. With `[embeddings]` installed, semantic vector search is available. An optional `sqlite-vec` package can further optimize vector storage — install it only if you need maximum query performance on large datasets.

## Verify Installation

Confirm everything is working:

<CodeBlock code={`hermes mnemosyne stats`} language="bash" />

This should print memory statistics (e.g. total memories, database size). You can also check the version:

<CodeBlock code={`hermes mnemosyne version`} language="bash" />
