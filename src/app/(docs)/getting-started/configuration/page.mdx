These keys live under `memory.mnemosyne` in Hermes Agent's `config.yaml`.

| Key | Default | Description |

| `sleep_threshold` | `50` | Working memory count before auto-sleep triggers |

| `profile_isolation` | `False` | Enable per-profile memory isolation via Mnemosyne banks. Each Hermes profile gets its own SQLite database under mnemosyne/data/banks/&lt;profile&gt;/. Default false for backward compatibility. |
| `shared_surface_path` | `data/shared/mnemosyne.db` | SQLite path for shared surface memories. Default is &lt;mnemosyne&gt;/data/shared/mnemosyne.db. |
| `shared_surface_read` | `False` | When true, mnemosyne_recall merges shared-surface results into private bank recall, tagging each result with its bank ('private' or 'surface'). Default false. |
| `skip_contexts` | `cron,flush,subagent,background,skill_loop` | Agent contexts where Mnemosyne should skip initialization. Comma-separated list. | `sync_roles` | `['user', 'assistant']` | Conversation roles to autosave in sync_turn(). List of role names: 'user', 'assistant'. Default ['user', 'assistant'] saves both. Identity signal capture is gated by user sync — excluding 'user' also disables identity extraction. Also configurable via MNEMOSYNE_SYNC_ROLES env var. |
