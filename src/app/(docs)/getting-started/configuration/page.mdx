# Configuration

Mnemosyne is configured via environment variables. All settings have sensible defaults so you can get started with zero configuration.

## Constructor Parameters

The only runtime parameters are passed directly to the constructor:

<CodeBlock code={`from mnemosyne import Mnemosyne

mem = Mnemosyne(
    session_id="my-agent",   # Scope working memories to this session
    db_path="./data/mem.db", # Custom database path (optional)
)
`} language="python" />

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `session_id` | `str` | `"default"` | Session identifier for scoping working memories |
| `db_path` | `str \| None` | `None` | Path to the SQLite database file |

## Environment Variables

### Storage & Memory Tiers

| Variable | Default | Description |
|----------|---------|-------------|
| `MNEMOSYNE_DATA_DIR` | — | Root data directory for database files |
| `MNEMOSYNE_WM_MAX_ITEMS` | `10000` | Maximum items in Working Memory |
| `MNEMOSYNE_WM_TTL_HOURS` | `24` | Time-to-live for Working Memory entries (hours) |
| `MNEMOSYNE_EP_LIMIT` | `50000` | Maximum items in Episodic Memory |
| `MNEMOSYNE_SLEEP_BATCH` | `5000` | Number of items to process per sleep cycle |
| `MNEMOSYNE_SP_MAX` | `1000` | Maximum scratchpad entries |
| `MNEMOSYNE_RECENCY_HALFLIFE` | `168.0` | Recency decay half-life in hours (default = 1 week) |

### Embeddings

Mnemosyne uses **BAAI/bge-small-en-v1.5** via fastembed (384 dimensions) for local embedding generation. No external API key is needed.

| Variable | Default | Description |
|----------|---------|-------------|
| `MNEMOSYNE_VEC_TYPE` | `int8` | Vector quantization type (e.g. `int8`, `float32`) |

### Local LLM (for Sleep Summarization)

The sleep cycle can use a local LLM to summarize grouped working memories before promoting them to episodic memory.

| Variable | Default | Description |
|----------|---------|-------------|
| `MNEMOSYNE_LLM_ENABLED` | `true` | Enable/disable LLM-based summarization |
| `MNEMOSYNE_LLM_MAX_TOKENS` | `256` | Maximum tokens for generated summaries |
| `MNEMOSYNE_LLM_N_THREADS` | `4` | Number of CPU threads for inference |
| `MNEMOSYNE_LLM_N_CTX` | `2048` | Context window size for the local model |
| `MNEMOSYNE_LLM_REPO` | — | Hugging Face repository for the local LLM |
| `MNEMOSYNE_LLM_FILE` | — | GGUF model filename within the repo |
| `MNEMOSYNE_LLM_BASE_URL` | — | External OpenAI-compatible API base URL (alternative to local LLM) |
| `MNEMOSYNE_LLM_API_KEY` | — | API key for the external LLM endpoint |
| `MNEMOSYNE_LLM_MODEL` | — | Model name for the external LLM endpoint |

<Callout type="info" title="No External Dependencies Required">
Mnemosyne runs entirely locally. Embeddings use fastembed with **BAAI/bge-small-en-v1.5** (384 dims), and the sleep cycle uses a local LLM via llamafile or an optional external OpenAI-compatible endpoint. No OpenAI API key is required for basic usage.
</Callout>
